# 后端技术架构

## 项目概览

- 项目名称: sports-lottery-backend
- 项目描述: 体彩投注记录管理系统后端
- 构建工具: Maven
- 后端语言: Java 8
- 主框架: Spring Boot 2.7.18
- 服务端口: 8080
- 应用入口: `src/main/java/com/sports/lottery/LotteryBackendApplication.java`
- 根包名: `com.sports.lottery`
- 当前激活配置: `pro`

## 技术栈

- Web 框架: Spring Boot Web
- 安全框架: Spring Security
- 认证方案: JWT Bearer Token
- 密码加密: BCrypt
- 参数校验: Spring Validation / Hibernate Validator
- ORM / 数据访问: MyBatis Plus
- 数据库: MySQL 8
- 数据库连接池: HikariCP
- 缓存组件: Spring Data Redis / Lettuce
- API 文档: springdoc-openapi / Swagger UI
- 工具库: Hutool
- 样板代码简化: Lombok
- JSON 处理: Jackson
- 测试依赖: Spring Boot Test、Spring Security Test

## 分层架构

- Controller 层:
    - 负责暴露 REST API、接收请求参数、读取登录用户信息、返回统一响应。
    - 主要类: `UserController`、`BettingRecordController`、`RecommendRecordController`。
- Service 层:
    - 负责业务规则、登录注册、投注记录处理、推荐记录处理和统计聚合。
    - 接口位于 `service`，实现位于 `service/impl`。
- Mapper 层:
    - 负责数据库访问。
    - 通用 CRUD 基于 MyBatis Plus `BaseMapper` 和 `ServiceImpl`。
    - 复杂统计查询通过 XML Mapper 编写 SQL。
- Entity 层:
    - 使用 MyBatis Plus 注解映射数据库表。
    - 主要实体: `User`、`BettingRecord`、`RecommendRecord`。
- DTO 层:
    - 负责请求参数、分页查询参数和校验规则。
    - 主要 DTO: `LoginRequest`、`RegisterRequest`、`BettingRecordRequest`、`BettingRecordQuery`、`PageQuery`。
- Common 层:
    - `Result<T>` 统一封装接口响应，包含 `code`、`message`、`data`、`timestamp`。

## 核心模块

### 用户模块

- 路由前缀: `/api/user`
- 主要功能:
    - 用户注册
    - 用户登录
    - 获取用户信息
    - 更新用户信息
    - 修改密码
- 技术实现:
    - 注册和改密使用 BCrypt 加密密码。
    - 登录成功后通过 `JwtUtil` 生成 JWT。
    - 用户状态字段用于控制账号是否启用。

### 投注记录模块

- 路由前缀: `/api/betting`
- 主要功能:
    - 分页查询投注记录
    - 新增投注记录
    - 批量新增投注记录
    - 更新投注记录
    - 删除投注记录
    - 批量删除投注记录
    - 用户投注统计
    - 按日期、联赛、投注类型统计
- 技术实现:
    - 通用 CRUD 使用 MyBatis Plus。
    - 分页使用 MyBatis Plus 分页插件。
    - 统计查询部分使用 `BettingRecordMapper.xml` 中的 SQL。
    - 操作记录前校验记录所属用户。

### 推荐记录模块

- 路由前缀: `/api/recommend`
- 主要功能:
    - 分页查询推荐记录
    - 新增推荐记录
    - 更新推荐记录
    - 删除推荐记录
- 技术实现:
    - 基于 MyBatis Plus `ServiceImpl` 实现通用 CRUD。
    - 支持按推荐日期和比赛信息筛选。
    - 使用当前登录用户 ID 做数据隔离。

## 认证与安全

- 安全配置类: `SecurityConfig`
- JWT 过滤器: `JwtAuthenticationFilter`
- JWT 工具类: `JwtUtil`
- 认证方式:
    - 客户端通过 `Authorization: Bearer <token>` 传递 JWT。
    - 过滤器解析 Token 后，将用户信息写入请求属性和 Spring Security 上下文。
- 放行接口:
    - `/api/user/register`
    - `/api/user/login`
    - `/v3/api-docs/**`
    - `/swagger-ui/**`
    - `/swagger-ui.html`
    - `/webjars/**`
    - HTTP OPTIONS 预检请求
- 会话策略:
    - 使用无状态会话 `SessionCreationPolicy.STATELESS`。
- 跨域:
    - `WebConfig` 中配置全局 CORS。

## 数据库设计

- 初始化脚本: `src/main/resources/sql/schema.sql`
- 数据库字符集: `utf8mb4`
- 主要数据表:
    - `user`: 用户表，包含用户名、密码、邮箱、昵称、头像、状态、创建时间、更新时间、最后登录时间。
    - `betting_record`: 投注记录表，包含用户、比赛日期、联赛、球队、投注类型、投注内容、赔率、金额、结果、盈利、备注等字段。
    - `recommend_record`: 推荐记录表，包含用户、推荐日期、比赛信息、推荐内容、推荐人、结果、创建时间、更新时间和逻辑删除字段。
- 关系:
    - `betting_record.user_id` 关联 `user.id`。
    - `recommend_record.user_id` 关联 `user.id`。
- MyBatis Plus 配置:
    - 开启下划线转驼峰。
    - 主键策略为自增。
    - 配置逻辑删除字段 `deleted`。
    - Mapper XML 位置为 `classpath*:mapper/*.xml`。


```bash
mvn clean package
```

```bash
java -jar target/lottery-backend-1.0.0.jar
```

## 架构特点

- 采用典型 Spring Boot 单体后端架构。
- 业务按 Controller、Service、Mapper、Entity、DTO 分层。
- 使用 JWT 实现前后端分离场景下的无状态认证。
- 使用 MyBatis Plus 降低 CRUD 代码量，复杂统计查询保留 XML SQL。
- 使用统一响应结构，便于前端统一处理接口结果。
- 使用 Swagger UI 暴露接口文档，便于联调。
