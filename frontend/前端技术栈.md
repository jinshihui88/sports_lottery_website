# 前端项目技术架构

## 项目概览

- 项目名称: sports-lottery-website
- 项目定位: 体育彩票投注记录管理系统前端
- 应用类型: 单页应用 SPA
- 主要功能: 登录鉴权、仪表板、推荐记录、投注记录、数据统计、日历视图、个人资料
- 构建入口: `index.html`、`src/main.ts`
- 应用根组件: `src/App.vue`

## 核心技术栈

- 前端框架: Vue 3
- 开发语言: TypeScript
- 构建工具: Vite 5
- 路由管理: Vue Router 4
- 状态管理: Pinia
- UI 组件库: Element Plus
- 图表方案: ECharts、vue-echarts
- 日历组件: FullCalendar Vue 3
- HTTP 请求: Axios
- 日期处理: dayjs
- 工具库: lodash-es
- 表单校验: vee-validate、yup
- 样式方案: CSS、Tailwind CSS、Element Plus 样式
- 代码检查: ESLint、eslint-plugin-vue、@typescript-eslint
- 类型检查: vue-tsc、TypeScript strict 模式

## 构建与工程化

- 包管理: npm，锁定文件为 `package-lock.json`
- 开发命令: `npm run dev`
- 构建命令: `npm run build`
- 预览命令: `npm run preview`
- 代码检查命令: `npm run lint`
- Vite 插件:
  - `@vitejs/plugin-vue`: 支持 Vue 单文件组件
  - `unplugin-auto-import`: 自动导入 Vue、Vue Router、Pinia、Element Plus API
  - `unplugin-vue-components`: 自动注册 Element Plus 组件
- 路径别名:
  - `@` 指向 `src`
- 开发服务器:
  - 默认端口: `3000`
  - `/api` 代理到本地后端 `localhost:8080`

## 目录结构

- `src/main.ts`: 应用初始化，注册 Pinia、Router、Element Plus
- `src/App.vue`: 应用根组件，承载 `router-view`
- `src/router/index.ts`: 路由配置与登录守卫
- `src/stores/`: Pinia 状态模块
  - `auth.ts`: 登录、登出、用户信息与 token 状态
  - `records.ts`: 投注记录、分页、统计、模拟数据操作
- `src/views/`: 页面级组件
  - `Login.vue`: 登录页
  - `Register.vue`: 注册页
  - `Dashboard.vue`: 数据仪表板
  - `RecommendRecords.vue`: 推荐记录管理
  - `Records.vue`: 投注记录管理
  - `Statistics.vue`: 数据统计分析
  - `Calendar.vue`: 日历视图
  - `Profile.vue`: 个人资料
- `src/components/`: 复用组件
  - `Layout.vue`: 后台布局、侧边栏、顶部栏
  - `RecordDialog.vue`: 投注记录新增/编辑弹窗
  - `BatchAddDialog.vue`: 批量新增投注记录弹窗
- `src/utils/`: 工具与请求封装
  - `api.ts`: Axios 实例、请求/响应拦截器、推荐记录接口
  - `format.ts`: 日期、金额、百分比、盈亏、胜率、ROI 格式化工具
- `src/types/index.ts`: 用户、投注记录、统计、图表、API 响应、分页、筛选参数类型
- `src/style.css`: 全局样式与 Tailwind 指令

## 路由架构

- 路由模式: `createWebHistory`
- 默认跳转: `/` 重定向到 `/dashboard`
- 公开路由:
  - `/login`
  - `/register`
- 登录后路由:
  - `/dashboard`
  - `/recommend-records`
  - `/records`
  - `/statistics`
  - `/calendar`
  - `/profile`
- 鉴权方式:
  - 路由 `meta.requiresAuth` 标识是否需要登录
  - 路由守卫从 `localStorage` 读取 `token`
  - 未登录访问受保护页面时跳转到 `/login`
  - 已登录访问登录/注册页时跳转到 `/dashboard`

## 状态与数据流

- 用户状态:
  - `useAuthStore` 保存 `user`、`token`、`isLoggedIn`
  - 登录成功后将 token 与用户信息写入 `localStorage`
  - 登出时清理 Pinia 状态与 `localStorage`
- 投注记录状态:
  - `useRecordsStore` 保存记录列表、加载状态、分页信息
  - 通过计算属性汇总投注次数、投注金额、中奖金额、净收益、胜率、平均投注金额
  - 当前投注记录模块主要使用本地模拟数据
- 推荐记录:
  - `RecommendRecords.vue` 通过 `src/utils/api.ts` 请求后端接口
  - 支持分页查询、新增、编辑、删除

## 请求架构

- Axios 基础地址:
  - 优先读取 `VITE_API_BASE_URL`
  - 未配置时使用 `/api`
- 请求拦截:
  - 从 `localStorage` 读取 token
  - 自动追加 `Authorization: Bearer <token>`
- 响应拦截:
  - 正常响应直接返回 `response.data`
  - `401` 时提示登录过期、清理本地登录信息并跳转登录页
  - `403`、`404`、`500` 和网络错误统一通过 Element Plus 消息提示
- 环境变量文件:
  - `.env.development`
  - `.env.production`
  - 已使用变量名: `VITE_APP_ENV`、`VITE_API_BASE_URL`、`VITE_APP_DEBUG`

## UI 与样式架构

- Element Plus 负责主要后台管理界面组件:
  - 布局容器、菜单、表单、表格、分页、弹窗、消息提示、卡片、标签、头像等
- Element Plus Icons 负责页面与菜单图标
- Tailwind CSS 已接入全局样式入口:
  - `@tailwind base`
  - `@tailwind components`
  - `@tailwind utilities`
- `tailwind.config.js` 扩展了:
  - `primary` 色板
  - `lottery` 业务色板
  - 中文字体族
  - 慢速动画

## 业务模块

- 登录模块:
  - 表单使用 Element Plus 校验规则
  - 登录接口通过 `loginApi` 调用后端
  - 登录成功后跳转仪表板
- 仪表板模块:
  - 展示投注总览、趋势图、类型分布、最近记录
  - 图表使用 vue-echarts
- 推荐记录模块:
  - 面向后端接口的 CRUD 页面
  - 支持推荐日期和比赛信息筛选
- 投注记录模块:
  - 支持搜索筛选、分页、新增、编辑、删除、批量添加、导出
  - 当前数据来源为 Pinia store 中的模拟数据
- 数据统计模块:
  - 提供趋势、分布、联赛胜率、投注时间分布、明细分析表格
- 日历模块:
  - 使用 FullCalendar 展示投注记录日历视图
  - 支持日期详情弹窗和盈亏图例

## 类型设计

- `User`: 用户信息
- `BetRecord`: 投注记录
- `Statistics`: 统计概览
- `ChartData`: 图表数据结构
- `ApiResponse<T>`: 后端统一响应结构
- `PaginationParams`: 分页参数
- `SearchParams`: 投注记录筛选参数

## 当前架构注意事项

- `tailwind.config.js` 的 `content` 当前配置为 `pages`、`components`、`app` 等路径，未包含本项目实际的 `src/**/*.{vue,js,ts}`，如果依赖 Tailwind 类名生成，建议后续补充 `src` 扫描路径。
- 根目录存在 `next.config.js`，但当前依赖与入口显示项目实际使用的是 Vite + Vue，不是 Next.js；该文件可能是历史遗留配置。
- `auth.ts` 中注册逻辑仍是模拟实现，未接入真实注册 API。
- `records.ts` 中投注记录仍主要使用本地模拟数据，推荐记录模块已接入 Axios 后端接口。
- `api.ts` 中部分推荐记录接口使用 `any`，后续可以补充更明确的请求与响应类型。
